Android build status
--------------------
- Added android/local.properties with SDK path:
  sdk.dir=C:\Users\Jayesh Sutar\AppData\Local\Android\Sdk
- Gradle initially required access to user .gradle cache and Android SDK files; elevated Gradle runs were used when needed.
- Build-Tools 36.0.0, platform android-36, and NDK 27.1.12297006 were found installed.
- assembleDebug did not complete successfully yet.
- Current Android blocker:
  RN 0.86 forces New Architecture by default. Native CMake builds for autolinked modules hit Android C++ runtime linker errors, especially in react-native-safe-area-context and react-native-gesture-handler, with missing std::__ndk1 / cxa / operator new/delete symbols.
- Attempted mitigation before stopping:
  - Added explicit CMake runtime linker arguments in android/app/build.gradle.
  - Added a root android/build.gradle subprojects block labeled "FlowTime native module C++ runtime linkage for RN 0.86 New Architecture builds on Windows."
  - Removed the unsupported newArchEnabled=false line from android/gradle.properties after RN warned that disabling New Architecture is no longer supported since RN 0.82.
- The final build was still failing in native C++ linking when the work was paused.
